# 乳腺癌知识库更新策略

## 版本管理

### 版本号规范

采用语义化版本 (Semantic Versioning):

```
主版本.次版本.修订版本
示例: 1.0.0, 1.1.0, 2.0.0
```

- **主版本 (Major)**: 重大结构调整、知识域重大变更
- **次版本 (Minor)**: 新增知识域、显著扩充现有内容
- **修订版本 (Patch)**: 错误修正、数据更新、格式调整

## 更新类型

### 1. 例行更新 (Quarterly)

每季度进行一次例行更新，主要包括：

- CSCO/NCCN等指南版本更新后的知识同步
- 新增重要临床研究证据
- 数据校验和错误修正

### 2. 重大版本更新 (Major Update)

每年或遇重大指南变更时进行：

- 新增知识域
- 三元组结构变更
- Schema定义更新
- 大规模数据扩充

### 3. 热修复 (Hotfix)

发现数据错误时及时修复：

- 错误的数值数据修正
- 逻辑错误修正
- 格式问题修复

## 数据质量标准

### 三元组质量要求

每个三元组必须满足：

1. **准确性**: 数据来源可靠，证据充分
2. **完整性**: 必需字段完整，无缺失
3. **一致性**: 与同领域其他三元组保持一致
4. **可追溯性**: 标注明确的来源和PMID

### 置信度评估标准

| 置信度范围 | 含义 | 适用情况 |
|------------|------|----------|
| 0.95-1.0 | 高度确定 | 权威指南明确推荐、大规模流行病学数据 |
| 0.85-0.94 | 较高确定 | 多项研究支持、专家共识 |
| 0.70-0.84 | 中等确定 | 单项高质量研究、小规模数据 |
| <0.70 | 仅供参考 | 初步研究、存在争议的观点 |

## 变更记录

所有更新必须记录在 CHANGELOG.md 中，格式：

```
## [版本号] - 日期

### Added
- 新增内容

### Changed
- 修改内容

### Fixed
- 修复内容

### Deprecated
- 弃用内容
```

## 审核流程

1. **自检**: 运行 `python3 scripts/tests/test_kb_format.py` 确保格式正确
2. **数据验证**: 检查新增内容的来源和准确性
3. **版本更新**: 更新 CHANGELOG.md 和元数据版本号
4. **Commit**: 提交变更并添加清晰的提交信息

## 贡献指南

欢迎提交 Pull Request 贡献知识：

1. 确保三元组格式符合 Schema 定义
2. 标注可靠的来源（优先使用指南和同行评审文献）
3. 运行测试确保格式正确
4. 提交时描述新增内容的来源和依据
